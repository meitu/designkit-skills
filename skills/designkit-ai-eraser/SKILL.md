# DesignKit AI Eraser

## Overview

AI 消除能力用于删除图片里不想保留的内容，属于 DesignKit 原子能力层。当前优先支持：

| 能力 | 操作标识 | 状态 | 描述 |
|------|---------|------|------|
| 去水印 / 去文字 | `eraser_watermark_text` | ✅ 可用 | 去掉水印、字幕、角标、画面文字 |
| 去眼镜 | `eraser_eyeglasses` | ✅ 可用 | 去掉人物眼镜 |
| 去路人 | `eraser_passerby` | ✅ 可用 | 去掉路人、背景行人、干扰人物 |
| 去照片反光 | `eraser_reflection` | ✅ 可用 | 去掉照片中的反光、眩光、镜面反射 |

## 意图识别

| 用户说法 | 路由到 |
|----------|--------|
| 去水印、消除水印、去文字、删字、移除字幕、去掉字样 | `eraser_watermark_text` |
| 去眼镜、去掉眼镜 | `eraser_eyeglasses` |
| 去路人、消除路人、移除路人、删除背景人物、去掉干扰人物 | `eraser_passerby` |
| 照片反光、去反光、消除反光、镜面反光 | `eraser_reflection` |

如果用户只说“帮我消除一下”“把多余东西去掉”，按上下文优先判断：

- 提到水印、文字、字样、字幕 → `eraser_watermark_text`
- 提到眼镜 → `eraser_eyeglasses`
- 提到路人、行人、背景人物 → `eraser_passerby`
- 提到照片反光、眩光、镜面反射 → `eraser_reflection`
- 无法判断时，再追问一次具体想去掉什么

## 对话追问策略

| 缺省信息 | 是否追问 | 追问话术 |
|---------|---------|---------|
| 没有图片 | 必须追问 | "请提供需要消除的图片（支持一张或多张，本地路径或 URL）。" |
| 只说“消除一下”但未说明对象 | 必须追问 | "你想去掉哪类内容？现在优先支持水印/文字、路人、眼镜、照片反光。" |
| 已经明确对象 | 不追问 | 直接执行 |

典型对话：

> 用户："把这张图里的水印去掉"
> Agent："好的，我来帮你把这张图里的水印和文字内容消除掉。" → 执行

> 用户："把人物脸上的眼镜去掉"
> Agent："好的，我来帮你把这张图里的人物眼镜去掉。" → 执行

## 执行原则

- 参数补齐后，按 `api/commands.json` 中已注册的能力执行
- 支持单张图，也支持多张图批量处理
- 不向用户展示底层接口、脚本路径、操作标识或实现细节
- 向用户只说明“我来帮你处理什么”，不要解释内部调用过程

## 结果处理

解析执行结果：

- `ok: true` → 从 `media_urls` 提取结果图 URL，用 `![结果图](url)` 展示给用户
- `ok: false` → 读取 `error_type` 和 `user_hint`，向用户展示可操作的指引

| `error_type` | 用户可见提示 |
|-------------|------------|
| `CREDENTIALS_MISSING` | 按 `user_hint` 引导用户配置 |
| `AUTH_ERROR` | 按 `user_hint` 引导用户核对 |
| `ORDER_REQUIRED` | 前往美图设计室获取美豆 |
| `PARAM_ERROR` | 按 `user_hint` 补齐参数 |
| `UPLOAD_ERROR` | 检查网络或换一张图片 |
| `API_ERROR` | 换一张图片试试 |

## Boundaries

本 skill 只处理明确的消除场景。以下情况不要直接承诺：

| 不做 | 处理方式 |
|------|---------|
| 需要精细框选或复杂局部修图 | 告知当前优先支持水印、文字、路人、眼镜、照片反光 |
